01 Mathematical Foundation — Bayes' Theorem
Bayes' Theorem

The Core Formula

Everything in Naive Bayes flows from one equation: the posterior probability of a hypothesis given observed evidence, expressed in terms of the likelihood, prior, and evidence.

P(A | B) = P(B | A) · P(A) / P(B) P(A|B) = Posterior · P(B|A) = Likelihood · P(A) = Prior · P(B) = Evidence (marginal)
Key insight: P(B) is the same for all classes — so we never compute it. We compare numerators only, which is why the formula becomes ∝ instead of =.
Naive Bayes Classifier

The Classification Formula

Apply Bayes' Theorem to classification. The "naive" step factorises the joint likelihood using the conditional independence assumption.

ŷ = argmax P(y) · ∏ᵢ P(xᵢ | y) "naive" assumption: P(x₁,x₂,...,xₙ | y) = ∏ P(xᵢ | y) — features independent given the class
P(y) — prior: how common is class y?
P(xᵢ|y) — likelihood per feature
argmax — pick the winning class
Worked Example — PlayTennis Dataset (from Session 1)

Manual Bayesian Calculation

Query: Outlook=Sunny, Temp=Cool, Humidity=Normal, Wind=True → Play?

STEP 1 — PRIORS
P(Yes) = 9/14 ← 9 yes rows P(No) = 5/14 ← 5 no rows
STEP 2 — LIKELIHOODS
P(Sunny|Yes) = 2/9 P(Cool|Yes) = 3/9 P(Normal|Yes) = 6/9 P(True|Yes) = 3/9 P(Sunny|No) = 3/5 P(Cool|No) = 1/5 P(Normal|No) = 2/5 P(True|No) = 3/5
STEP 3 — POSTERIOR
# unnormalized scores P(Yes|W) ∝ 9/14·2/9·3/9·6/9·3/9 = 0.0053 P(No|W) ∝ 5/14·3/5·1/5·2/5·3/5 = 0.0206 → Predict: No (0.0206 > 0.0053)

03 Variant Deep Dives
Gaussian NB

Continuous Features

At training time, compute μ (mean) and σ² (variance) per feature per class. At inference, plug x into the Gaussian PDF to get the likelihood.

P(xᵢ|y) = 1/√(2πσ²) · exp( −(x−μ)² / 2σ² ) σ² serves as built-in regularisation — variance can never be zero if data varies
Why no Laplace smoothing? GaussianNB uses a continuous PDF — the output is never exactly 0 as long as σ > 0. Laplace smoothing is for discrete counts only.
from sklearn.naive_bayes import GaussianNB gnb = GaussianNB() gnb.fit(X_train, y_train) # Iris dataset → ~97% accuracy
Multinomial NB

Word Counts / Frequencies

Ideal for text. Estimates word probability as smoothed count ratio. The update rule works in log-space for numerical stability.

P(wᵢ|y) = (count(wᵢ,y) + α) / (total_words_y + α·|V|) α = Laplace smoothing param · |V| = vocabulary size
ŷ = argmax [ log P(y) + Σᵢ xᵢ · log P(wᵢ|y) ] log-space update rule — avoids float underflow
Pipeline([('vec', CountVectorizer()), ('clf', MultinomialNB(alpha=1.0))])
Bernoulli NB

Word Presence (0 or 1)

Unlike Multinomial, Bernoulli explicitly penalises absent words. Each position in the binary vector contributes — whether the word is present or absent.

P(xᵢ|y) = P(i|y)^xᵢ · (1−P(i|y))^(1−xᵢ) xᵢ=1 → contributes P(i|y) · xᵢ=0 → contributes 1−P(i|y)
# binary=True forces 0/1 per word Pipeline([('vec', CountVectorizer(binary=True)), ('clf', BernoulliNB(alpha=1.0))])
Key difference from Multinomial: repeating "free free free" in an email has NO extra effect. Only presence matters.
Categorical NB

Discrete Label Features

Each feature has a categorical distribution with its own number of categories nⱼ. Requires OrdinalEncoder first — sklearn expects integer-encoded categories.

P(Outlook=Sunny|No) = (count+α) / (N_No + α·nⱼ) nⱼ = number of categories for feature j · computed per-feature
enc = OrdinalEncoder() X_enc = enc.fit_transform(X_cat) clf = CategoricalNB(alpha=1.0) clf.fit(X_enc, y)
Complement NB — the imbalanced-data fix

Learn What a Class is NOT

Instead of P(xᵢ|class=c), ComplementNB estimates P(xᵢ|class≠c) — the probability of the feature in all other classes. This uses far more training data per weight, reducing variance on minority classes.

COMPLEMENT WEIGHT
w_ci = log( (α + Σ_{j: yj≠c} xji) / (α + Σ_{j: yj≠c} Σ_k xjk) ) normalised by L2 norm · minimise complement score, not maximise posterior
WHEN TO USE
  • 1Imbalanced text data — minority classes underrepresented
  • 2News categorisation — many categories, uneven volumes
  • 3Document routing — some topics much rarer than others
  • 4Empirically outperforms MultinomialNB on text benchmarks

05 Laplace Smoothing — Zero-Frequency Problem
The Problem

One Zero Kills Everything

If a word has never been seen with a class during training, its count is 0, making the entire product collapse — no matter how strong the other 999 features are.

# Word "blockchain" never in training spam P("blockchain"|spam) = 0/total = 0 # Entire posterior → 0 P(spam|email) = P(spam) × 0 × ... = 0.0 # ↑ Catastrophically wrong! All evidence ignored.
The Solution

Add-α Smoothing

Add α (typically 1 for Laplace, <1 for Lidstone) to every feature count before computing probabilities. This guarantees no probability is ever zero.

P(wᵢ|y) = (count(wᵢ,y) + α) / (count(y) + α·|V|) +α numerator → every word seen at least α times +α·|V| denominator → valid probability distribution (sums to 1) |V| = vocabulary size · α=1 → uniform Dirichlet prior
Bias-variance view: higher α = more smoothing = higher bias, lower variance. Tune alpha as a hyperparameter — test values like 0.001, 0.01, 0.1, 0.5, 1.0, 2.0, 5.0 on validation data.
Alpha Tuning (from IMDB Notebook)

Optimal Alpha on MultinomialNB

alphas = [0.001, 0.01, 0.1, 0.5, 1.0, 2.0, 5.0, 10.0] for alpha in alphas: clf = MultinomialNB(alpha=alpha) clf.fit(X_train, y_train) acc = accuracy_score(y_test, clf.predict(X_test))
  • α→0No smoothing — zero-frequency problem remains. Very low accuracy on unseen vocabulary.
  • α=1Laplace smoothing — uniform prior. Good default for most text tasks. Mathematically clean.
  • 0<α<1Lidstone smoothing — fractional prior. Can outperform α=1 on large datasets with rich vocabulary.
  • α→∞Over-smoothed — all words equally likely → useless classifier. High bias.
06 Log Probabilities — Numerical Stability
The Problem

Floating-Point Underflow

Multiplying hundreds of probabilities together (each <1) creates values smaller than float64 can represent, causing silent rounding to 0.0.

# 300 features, each P ≈ 0.01 0.01 ** 300 = 10⁻⁶⁰⁰ # float64 minimum ≈ 5×10⁻³²⁴ # 10⁻⁶⁰⁰ underflows → 0.0 silently # All classes become 0 → undefined argmax
Real consequence: without log-space, sklearn NB would silently give wrong predictions on high-dimensional sparse data — e.g. text with 10,000-word vocabularies.
The Solution

Work in Log-Space

Convert products to sums using log. Argmax is preserved because log is strictly monotonic. All sklearn NB implementations use log internally.

log P(y|x) ∝ log P(y) + Σᵢ log P(xᵢ|y) log(A·B) = log A + log B → multiplication becomes addition
# sklearn stores log probabilities gnb.class_log_prior_ # log P(y) mnb.feature_log_prob_ # log P(xᵢ|y) # Prediction: argmax of log-posteriors # mathematically identical to argmax of posteriors
Caveat: log(0) = −∞. You MUST apply Laplace smoothing before taking logs — otherwise absent words produce −∞ and break summation.
07 What If Data Is Not Gaussian? (Session 1)

Five Options When GaussianNB Assumptions Fail

  • 1Data Transformation — apply log, square root, or reciprocal to make features more normally distributed before using GaussianNB.
  • 2Alternative Distributions — modify NB to assume Poisson, Exponential, etc. if you know the true distribution. Requires custom implementation.
  • 3Discretization / Binning — convert continuous features to categorical bins (equal-width, equal-frequency, or k-means). Then use CategoricalNB. Trade-off: information loss.
  • 4Kernel Density Estimation (KDE) — non-parametric density estimation. Estimates P(x|y) without assuming any distribution. More accurate but slower to evaluate.
  • 5Switch Models — Decision Trees, Random Forests, SVMs make no distribution assumptions and often outperform GaussianNB on non-normal continuous data.

09 Out-of-Core (Incremental) Learning
Why NB is Ideal for Streaming

partial_fit() API

NB only stores summary statistics (class priors + per-class feature stats), not raw data. Memory usage is O(classes × features) — constant regardless of dataset size.

# Train in 1000-row chunks gnb = GaussianNB() classes = np.unique(y) for i in range(0, len(X), 1000): gnb.partial_fit( X[i:i+1000], y[i:i+1000], classes=classes # required first call ) # Result = same as batch fit()
Properties

What Makes it Work

  • StatsSufficient statistics only — NB needs counts (or μ/σ²). Each batch updates these accumulators.
  • MemoryO(classes × features) — RAM doesn't grow with dataset size. Process terabytes with megabytes of RAM.
  • ExactMathematically equivalent to batch — streaming result is identical to training on all data at once.
  • APIWorks on GaussianNB MultinomialNB BernoulliNB — all support partial_fit()

11 Strengths & Weaknesses
✓ Strengths

Where NB Shines

  • SpeedO(n·d) training and prediction — linear in samples × features. Scales to millions of docs.
  • DataWorks with small training sets — independence assumption reduces sample complexity dramatically.
  • DimsHandles high-dimensional sparse data — text with 100k+ features is no problem.
  • IncrOnline/incremental learning — partial_fit() updates statistics; no need to re-train from scratch.
  • MissHandles missing values elegantly — just skip the missing feature in the product; no imputation needed.
  • ReadInterpretable — inspect feature_log_prob_ to see which words drive predictions.
✗ Weaknesses

Where NB Falls Short

  • DepIndependence assumption almost always wrong — "buy" and "cheap" co-occur in spam; NB ignores this correlation.
  • ProbPoor probability calibration — posteriors are often overconfident. Don't trust raw P(y|x) scores for risk decisions.
  • DistGaussian assumption may fail — multimodal, skewed, or bounded continuous features violate normality.
  • XORCannot learn feature interactions — XOR-like patterns (where the combination matters, not individual features) are invisible to NB.
  • ScaleOutperformed on large datasets — SVM, Random Forest, Neural Networks capture correlations that NB ignores.
  • ZeroZero-frequency problem — discrete variants require smoothing; forgetting α can silently destroy accuracy.
12 Top Interview Questions & Answers
Q1What is the "naive" assumption and when does it break?
Features are conditionally independent given the class: P(x₁,...,xₙ|y) = ∏ P(xᵢ|y). It breaks when features are correlated — e.g. word co-occurrences ("New York"), pixel values in images, or correlated sensor readings.
Q2Why does NB work despite the naive assumption being wrong?
For classification, only the ranking of classes matters, not absolute probabilities. Even with wrong absolute posteriors, the ordering is often preserved — especially in high-dimensional sparse data where the independent signal is still dominant.
Q3Why use log probabilities in Naive Bayes?
To prevent numerical underflow. Multiplying hundreds of small probabilities (each <1) produces values below float64 minimum (~5×10⁻³²⁴), silently rounding to 0. Log converts products to sums: log(A·B) = log A + log B. Since log is monotonic, argmax is unchanged — predicted class is identical.
Q4Why is Laplace smoothing NOT used in Gaussian NB?
GaussianNB uses a continuous PDF, not discrete counts. The Gaussian PDF is never exactly zero as long as σ > 0. Zero-frequency has no meaning in continuous space — there are infinitely many possible x values. The variance σ² serves the same regularisation role that α serves in discrete variants.
Q5When would you choose each NB variant?
GaussianNB — continuous features (age, height, sensor readings). MultinomialNB — word counts or TF-IDF, long documents. BernoulliNB — binary word presence, short texts. CategoricalNB — nominal label features (gender, colour, grade). ComplementNB — imbalanced text classification.
Q6Can Naive Bayes handle missing values?
Yes — elegantly. Simply skip the missing feature in the product. Because each term is independent, omitting one doesn't invalidate the others. No imputation needed. This is a genuine advantage over most other classifiers.
Q7What is Complement NB and when should you use it?
ComplementNB estimates P(feature|NOT class) instead of P(feature|class). This uses all non-class samples per weight, giving much more training data for minority classes and reducing variance. Use it whenever your text classification data has imbalanced class distribution.
Q8What makes Naive Bayes suitable for out-of-core learning?
NB only needs to maintain sufficient statistics (counts or μ/σ²), not raw data. partial_fit() updates these statistics from each mini-batch. Memory is O(classes × features) — constant regardless of dataset size. Streaming result is mathematically identical to batch training.
13 Master Summary
Bayes' Theorem
P(A|B) = P(B|A)·P(A)/P(B) — foundation. P(B) cancels in classification so we compare numerators only.
Naive Assumption
Features are conditionally independent given class: P(x|y) = ∏ P(xᵢ|y). Rarely true, often good enough.
Gaussian NB
Continuous features → learns μ, σ² per class per feature → uses Gaussian PDF. No Laplace smoothing needed.
Multinomial NB
Word counts/TF-IDF → best for long documents. P(w|y) = (count+α)/(total+α·|V|). Requires Laplace smoothing.
Bernoulli NB
Binary 0/1 features → penalises absent words. Better for short texts. P(xᵢ|y) uses both presence and absence terms.
Categorical NB
Nominal labels → requires OrdinalEncoder → CategoricalNB. For mixed data, combine with GaussianNB via log-posterior sum.
Complement NB
Uses P(feature|¬class) → less variance on minority classes → better on imbalanced text classification.
Laplace Smoothing
Add α to counts before computing P(x|y). Prevents zero-frequency collapse. α=1 (Laplace), 0<α<1 (Lidstone). Tune as hyperparameter.
Log Probabilities
Convert ∏ P(xᵢ|y) to Σ log P(xᵢ|y). Prevents float underflow. Monotonic → argmax unchanged. sklearn uses internally.
Out-of-Core
partial_fit() updates sufficient statistics per batch. Memory = O(classes × features). Identical result to batch fit().
Non-Gaussian Data
Options: transform data, use KDE, discretize → CategoricalNB, use mixed NB, or switch to Decision Tree / Random Forest.
NLP Best Practice
Preprocess: strip HTML → lowercase → remove special chars → remove stopwords → stem → CountVectorizer. Prefer Multinomial or Bernoulli over Gaussian for text.
