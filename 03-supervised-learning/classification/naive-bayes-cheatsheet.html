Naive Bayes

CLASSIFICATION

Probabilistic · Generative

01 · Core Idea

What is it?

A probabilistic classifier built on Bayes' Theorem. Given a set of features, it picks the class with the highest posterior probability. "Naive" because it assumes all features are conditionally independent given the class — a bold simplification that works surprisingly well.

Ĉ = argmax P(C) · ∏ P(xᵢ | C) posterior ∝ prior × product of likelihoods

Key Terms

P(C) — Prior probability of class C
P(xᵢ|C) — Likelihood of feature given class
P(C|x) — Posterior (what we want)
argmax — Pick the class that maximises
Why it's fast: No need to compute P(x) — it's the same for all classes, so we just compare numerators.

03 · Variant Deep Dives

Gaussian NB

Continuous Features

Assumes feature values follow a Normal distribution within each class. At training time, compute μ and σ² per feature per class. At inference, plug into the Gaussian PDF.

P(x|C) = (1/√2πσ²) · exp( −(x−μ)² / 2σ² )
μ — class feature mean
σ² — class feature variance

Multinomial NB

Word Counts

Ideal for text. Uses how often each word appears. Estimated as word count in class divided by total words in class.

Document: "free free money offer" → free=2 money=1 offer=1 P("free"|spam) = count(free,spam) + 1 ───────────────────── total_words_spam + V

Bernoulli NB

Word Presence

Only cares about whether a word appears — not how many times. Generates a binary feature vector over the entire vocabulary. Explicitly penalises absent words too.

Vocab: free, money, meeting Doc: "free meeting" Vector: free=1 money=0 meeting=1 # 0 entries still contribute to likelihood

Categorical NB

Discrete Labels

Each feature takes one of a fixed set of discrete values (not numeric). Estimate P(category | class) directly from frequency counts.

Weather → Play Tennis? P(Sunny | Play=No) = 3/5 P(Rainy | Play=Yes) = 2/9 P(Cloudy | Play=Yes) = 4/9 # One table per feature

05 · Laplace Smoothing

The Problem

Zero Probability

If a word never appeared in training for a class, P(word|class) = 0. Since we multiply likelihoods, one zero kills the entire product, regardless of all other evidence.

# Word "blockchain" never seen in spam: P("blockchain" | spam) = 0 P(spam | email) = P(spam) × 0 × ... = 0 # ↑ Catastrophically wrong!

The Solution

Add-1 Smoothing

Add 1 to every word count (α=1 is "Laplace"; α<1 is "Lidstone"). Guarantees no probability is ever zero.

count(word, C) + 1 total_words_C + V
+1 in numerator — pretend each word seen once
+V in denominator — V = vocabulary size
Maintains valid probability distribution
06 · Strengths & Weaknesses

✓ Strengths

Extremely fast — linear in features and training samples. Scales to millions of docs effortlessly.
Handles high-dimensional sparse data well — text classification is the perfect use case.
Works with very little training data — independence assumption reduces sample complexity.
Interpretable — you can inspect which features drive predictions for each class.
Online learning — update counts incrementally as new data arrives.

✗ Weaknesses

Independence assumption is almost always wrong — feature correlations are ignored ("buy" and "cheap" co-occur heavily in spam).
Poor probability calibration — predicted probabilities are often overconfident; don't trust raw scores.
Continuous features require Gaussian assumption — fails if the real distribution is multimodal or skewed.
Zero-frequency problem — must use smoothing; raw counts collapse on unseen words.
07 · High-Frequency Interview Q&A
What is the "naive" assumption and when does it break?
Features are conditionally independent given the class. It breaks when features are correlated — e.g. word co-occurrences in text ("New York"), pixel values in images, or correlated sensor readings.
Why does NB still work well despite the naive assumption?
For classification (not probability estimation), the ranking of classes is what matters. Even with wrong absolute probabilities, the ordering is often correct enough to beat more complex models — especially in high-dimensional sparse settings.
Multinomial NB vs Bernoulli NB — which is better for text?
Multinomial generally wins for longer documents where frequency carries signal. Bernoulli can win for short texts / keyword detection where word count matters less than vocabulary overlap.
What does Laplace smoothing do and why α=1?
It adds α (typically 1) to every count before computing probabilities, ensuring no probability is zero. α=1 is the maximum a posteriori estimate under a uniform Dirichlet prior. Use α<1 (Lidstone smoothing) if your data is large enough.
Can Naive Bayes do regression?
No — it is inherently a classification algorithm. It predicts a discrete class label, not a continuous value. For regression, use Gaussian process, linear regression, etc.
How does Naive Bayes handle missing values?
Elegantly — just skip the missing feature in the product. Because each term is independent, omitting one doesn't invalidate the others. No imputation needed.
